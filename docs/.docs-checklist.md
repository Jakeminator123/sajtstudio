# Dokumentationschecklista

**Öppna denna fil när du gör ändringar för att komma ihåg att uppdatera dokumentationen.**

## Snabbchecklista

### Komponent-ändringar

- [ ] Ny komponent skapad? → Uppdatera `SITE_STRUCTURE.md`
- [ ] Komponent flyttad? → Uppdatera filvägar i `SITE_STRUCTURE.md` och `MOBILE_DESKTOP_INDEX.md`
- [ ] Mobil/desktop-funktionalitet ändrad? → Uppdatera `MOBILE_DESKTOP_INDEX.md`
- [ ] Kommentarer i kod lagda till? → Lägg till tydliga kommentarer som markerar mobil vs desktop

### Responsive-ändringar

- [ ] Nya responsive patterns? → Uppdatera `RESPONSIVE_DESIGN_GUIDE.md` och `responsiveUtils.ts`
- [ ] Breakpoints ändrade? → Uppdatera `RESPONSIVE_DESIGN_GUIDE.md` och `MOBILE_DESKTOP_INDEX.md`

### Struktur-ändringar

- [ ] Filer flyttade? → Uppdatera alla filvägar i dokumentationen
- [ ] Mappstruktur ändrad? → Uppdatera `SITE_STRUCTURE.md` och `.cursorrules`
- [ ] Ny mapp/kategori? → Uppdatera `SITE_STRUCTURE.md` och `.cursorrules`

## Snabbreferens

| Ändring                      | Fil att uppdatera                                  |
| ---------------------------- | -------------------------------------------------- |
| Ny komponent                 | `SITE_STRUCTURE.md`                                |
| Komponent flyttas            | `SITE_STRUCTURE.md`, `MOBILE_DESKTOP_INDEX.md`     |
| Mobil/desktop-funktionalitet | `MOBILE_DESKTOP_INDEX.md` + kommentarer i kod      |
| Responsive patterns          | `RESPONSIVE_DESIGN_GUIDE.md`, `responsiveUtils.ts` |
| Strukturändring              | `SITE_STRUCTURE.md`, `.cursorrules`                |

## Workflow

1. Gör din kodändring
2. Öppna denna checklista
3. Kolla vad som behöver uppdateras
4. Uppdatera dokumentationen
5. Commit allt tillsammans

**Tips:** Uppdatera dokumentationen i samma commit som kodändringen så glömmer du inte bort det.
